| Flag | Description |
|------|-------------|
| `--state-dir <path>` | Override the default state directory (default: `~/.local/state/arok`) |

## arok doctor

Validates installation and prints key diagnostics.

```
version          v0.3.0
update_available up to date
state_dir        /home/user/.local/state/arok
database         ok
copilot_hook     ok
```

---

## arok analyze overview

Aggregate analytics across all sessions.

**Flags:**
| Flag | Default | Description |
|------|---------|-------------|
| `--since <duration>` | all time | Time window, e.g. `24h`, `168h`, `720h` |
| `--limit <n>` | 5 | Max rows in each grouped breakdown |

**Output sections:** summary counts → top hosts → top repos → top branches → top models → sessions missing final totals (if any)

---

## arok query sessions

List sessions or inspect a single session.

**Flags:**
| Flag | Default | Description |
|------|---------|-------------|
| `--latest <n>` | 20 | How many sessions to list |
| `--session-id <id>` | — | Print full JSON for one session |
| `--host <name>` | — | Filter by hostname |
| `--repo <url>` | — | Filter by git remote URL |
| `--branch <name>` | — | Filter by branch name |
| `--since <duration>` | — | Only sessions newer than now-duration |

When any filter flag is set, output appends an aggregate totals line:
```
totals  sessions=N  input=X  output=Y  cache_read=Z  cache_write=W  reasoning=R
```

**Columns:** SESSION · HARNESS · STATE · USAGE_SOURCE · HOST · BRANCH · WORKTREE · INPUT · OUTPUT · ENDED_AT

**capture_state values:**
- `final` — complete, authoritative token data
- `provisional` — hook fired but metrics not yet confirmed
- `best_effort` — reconciliation exhausted; best data available

## arok update

Self-updates the binary from the latest GitHub release.

```bash
arok update   # or: arok upgrade
```

---

## Token Field Glossary

| Field | Meaning |
|-------|---------|
| `input` / `total_input_tokens` | Tokens sent to the model (prompt + context) |
| `output` / `total_output_tokens` | Tokens generated by the model |
| `cache_read` | Prompt cache hits (reduce effective input cost) |
| `cache_write` | Tokens written to prompt cache |
| `reasoning` | Internal chain-of-thought tokens (extended thinking models) |

---

## Duration Format

All `--since` flags accept Go duration strings:

| Value | Meaning |
|-------|---------|
| `24h` | Last 24 hours |
| `168h` | Last 7 days |
| `720h` | Last 30 days |
| `2160h` | Last 90 days |
